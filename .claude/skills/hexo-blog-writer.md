# Hexo Blog Writer

Create Hexo blog posts documenting GitHub repositories with proper front matter and formatting.

## When to Use

- User wants to create a new blog post about a GitHub repository
- User wants to document a library/tool from GitHub
- User requests a guide/tutorial for a GitHub project

## Prerequisites

- Hexo project structure
- Node.js with hexo installed locally

## Workflow

### Step 1: Invoke professor-synapse Skill

**IMPORTANT**: First invoke the `professor-synapse` skill to leverage its tech-doc-writer agent for high-quality documentation generation.

```text
Skill: professor-synapse
```

The professor-synapse skill will:
- Orchestrate expert agents for documentation writing
- Ensure professional structure and formatting
- Generate comprehensive guides with proper sections

### Step 2: Verify Hexo Configuration

Check `_config.yml` for proper post naming format:

```yaml
new_post_name: :year-:month-:day-:title.md
```

If not set correctly, update it to include date prefix.

### Step 3: Create Post via Hexo CLI

**IMPORTANT**: Must use `hexo new` command, NOT manual file creation.

```bash
cd <project-root>
node ./node_modules/hexo/bin/hexo new "<post-title>"
```

This ensures:
- Correct front matter generation
- Proper file naming
- Hexo's internal state is updated

### Step 4: Fetch Repository Content

Use GitHub MCP tool to get README:

```text
mcp__plugin_ecc_github__get_file_contents
  owner: <repo-owner>
  repo: <repo-name>
  path: README.md
```

### Step 5: Write Blog Content

Use professor-synapse's tech-doc-writer agent output to structure the content:

1. **Preserve front matter** - Keep generated title, date
2. **Add relevant tags** - Based on repository topic
3. **Structure content** (following professor-synapse output):
   - 简介 (Introduction)
   - 核心功能/概念 (Core features/concepts)
   - 安装使用 (Installation & Usage)
   - 代码示例 (Code examples if applicable)
   - 总结 (Summary)

### Step 6: Verify

- File exists in `source/_posts/`
- File name format: `YYYY-MM-DD-title.md`
- Front matter is valid YAML

## Template

```markdown
---
title: <title>
date: <auto-generated>
tags:
  - <tag1>
  - <tag2>
categories:
  - <category>
---

## <Title> 入门指南

## 简介

Brief introduction with GitHub link.

## 核心功能/概念

Main features or concepts explained.

## 安装使用

Installation steps.

## 使用示例

Code examples.

## 总结

Wrap up.
```

## Common Issues

### Node.js Path Issues

If `hexo` command fails with module not found:

```bash
# Use direct node execution
node ./node_modules/hexo/bin/hexo new "<title>"
```

### File Naming Without Date

If `new_post_name` is `:title.md`, update `_config.yml`:

```yaml
new_post_name: :year-:month-:day-:title.md
```

## Example

User request: "Create a blog post about <https://github.com/user/awesome-lib>"

1. **Invoke professor-synapse skill** for documentation generation
2. Check `_config.yml` for `new_post_name`
3. Run: `node ./node_modules/hexo/bin/hexo new "awesome-lib入门指南"`
4. Fetch: `mcp__plugin_ecc_github__get_file_contents(owner: "user", repo: "awesome-lib", path: "README.md")`
5. Use professor-synapse output to write content to created file
